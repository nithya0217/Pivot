CLIENT UNDERSTANDING:
We are building a digital publishing platform where authors can publish articles with tags and users can consume content. 
I have chosen Track B (Algorithmic Model) with a Negative Bias (Contrarian/Diversity Routing).
The system will track user interactions with tags and recommend content that sometimes challenges the user's preferences by introducing opposite or diverse topics, improving content discovery and avoiding echo chambers.



CREATIVE FREEDOMS:

- Product Name: "Pivot"
- UI Design: Clean card-based layout similar to Medium
- Tag System: Each article can have up to 5 tags
- Opposite Tag Mapping:
  Example:
  - "Technology" ↔ "Philosophy"
  - "Politics-Left" ↔ "Politics-Right"
  - "Fitness" ↔ "Relaxation"
- Feed Mix Strategy:
  - 70% user-interest content
  - 30% diverse/opposite content
- Dark mode UI support
- Simple like/bookmark system



ASSUMPTIONS:

- Users must log in to get a personalized feed
- Authors must log in to create articles
- Each article must have at least 1 tag
- Maximum 5 tags per article
- User interaction includes clicks, likes, and reading time
- Feed refresh happens on every login
- Opposite tags are predefined manually (not AI-based)
- MVP will not include comments initially



FEATURE LIST:

1. User Authentication
   Users can register, login, and logout securely.

2. Author Article CRUD
   Authors can Create, Read, Update, Delete articles.

3. Tagging System
   Articles are categorized using tags for filtering and recommendations.

4. Feed Algorithm
   Personalized feed based on user interaction history.

5. Diversity Engine
   Introduces opposite-tag content into feed to avoid echo chambers.

6. Interaction Tracking
   Tracks user clicks, likes, and reading time.

7. Bookmark System
   Users can save articles for later reading.

8. Search & Filter
   Users can search articles by tags or keywords.

9. Responsive UI
   Works on mobile and desktop.

10. Admin Tag Mapping
    Admin defines opposite tag relationships.
